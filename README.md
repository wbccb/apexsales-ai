# 🚀 销售 AI 闭环系统 (Sales AI Closed-Loop System)

## 📖 项目简介
本项目是一个面向销售场景的 **AI 全链路工作流系统**。旨在通过人工智能技术，打通从“客户沟通录音”到“业务系统演示”，再到“合同自动生成”的完整闭环。系统以快速演示和高可用性为目标，帮助销售人员在与客户沟通后，一键生成结构化需求、可交互的 POC（概念验证）前端页面以及精准的报价合同。

## ✨ 核心特性 (Features)

### 🎙️ 1. 智能语音采集与识别 (ASR & Voiceprint)
* **前端端点检测 (VAD)**：智能识别说话停顿，自动触发语音切片上传，降低流式并发压力。
* **角色分离 (1v1 判别)**：支持销售声纹注册与验证，智能分离“销售”与“客户”的对话内容。
* **高可用转写**：将离散的音频切片转写为准确的中文逐字稿，支持会话上下文拼接。

### 📚 2. RAG 知识库与检索增强 (P0 级特性)
* **文档管理**：提供独立的右侧边栏知识库管理界面，支持产品文档、报价单（PDF）的上传与归档。
* **智能切片与向量化**：对 PDF 进行自动切片、计算 Embedding 并持久化存储。
* **精准溯源**：在生成 PRD 和合同时，系统会自动检索 TopK 相关片段注入 Prompt，确保所有报价和产品特性均**有据可查，杜绝 AI 幻觉**。

### 📝 3. 会后总结与 PRD 编辑
* **结构化总结**：基于语音逐字稿和 RAG 检索结果，LLM 自动生成清晰的 Markdown 格式 PRD（需求文档）。
* **人工介入**：前端提供沉浸式 PRD 编辑器，允许销售在最终生成前对需求进行人工校阅和保存。

### 💻 4. 动态 POC 生成与零门槛分享 (独立 H5 方案)
* **一键生成代码**：基于 PRD 需求，LLM 自动编写一份完整的、可独立运行的 HTML5 前端页面代码（依赖项通过 CDN 引入）。
* **即时预览**：前端通过 `iframe` 安全沙箱实时渲染生成的 POC 页面。
* **持久化短链分享**：生成专属 UUID 分享链接。客户打开链接无需任何输入，即可直接体验演示页面。

### 📄 5. 自动化合同导出
* **结构化报价注入**：基于当前定稿的 PRD 与 RAG 知识库检索结果，LLM 自动生成包含准确单价、数量、总价、付款节点等字段的规范合同。
* **PDF 导出**：支持将生成的合同内容一键转换为 PDF 格式并提供稳定下载链接。

### ⚙️ 6. 灵活的配置与持久化
* **LLM 动态配置**：左侧边栏提供 LLM 参数配置面板。支持从 `.env` 读取默认配置，用户修改后可存入数据库（优先级：数据库 > `.env`）。
* **核心数据落库**：用户、会话、逐字稿、PRD、POC、合同、声纹数据等核心业务实体全面实现数据库持久化，服务重启数据不丢失。

## 🛠️ 技术闭环流转图

1. **采集阶段**：麦克风监听 -> 切片上传 (`audio/wav`) -> 携带 `session_id`。
2. **转写阶段**：ASR 接口 -> 提取文本与说话人标签 -> 存储至数据库。
3. **分析阶段**：获取逐字稿 -> 触发 RAG 检索知识库 -> LLM 生成总结 -> 渲染 PRD 编辑器。
4. **演示阶段**：确认 PRD -> LLM 编写独立 HTML 文件 -> `iframe` 渲染 -> 生成 UUID 独立分享页。
5. **成交阶段**：综合 PRD 与 RAG 检索信息 -> LLM 生成最终合同 -> 导出 PDF 下载。

## 🔌 接口全览 (API Endpoints)

| 功能模块 | 接口路径 | 方法 | 说明 |
| :--- | :--- | :--- | :--- |
| **系统** | `/health` | `GET` | 健康检查 |
| **语音声纹** | `/voice/register` | `POST` | 注册销售声纹 |
| | `/voice/verify` | `POST` | 验证说话人身份 |
| **转写** | `/asr/transcribe` | `POST` | 音频切片转写 |
| **会话记录** | `/session/{session_id}/utterances` | `GET` | 获取会话逐字稿 |
| | `/session/{session_id}/summary` | `POST` | 生成会话总结与 PRD |
| **PRD 管理** | `/prd/{prd_id}/save` | `POST` | 保存修改后的 PRD |
| **POC 演示** | `/prd/{prd_id}/poc` | `POST` | 基于 PRD 生成单文件 HTML POC |
| | `/poc/{share_uuid}` | `GET` | 根据短链获取 POC 内容并渲染 |
| **合同导出** | `/contract/{prd_id}` | `POST` | 生成最终报价合同 (结合 RAG) |
| | `/contract/{contract_id}/download`| `GET` | 下载合同 PDF |

*(注：RAG 相关的文档上传与检索接口依据具体实现路径进行补充)*

## 🚀 快速开始 (Getting Started)

### 1. 环境准备
确保你的本地环境已安装以下依赖：
* Node.js (v16+ 推荐) / Python (后端相关)
* 数据库 (MySQL/PostgreSQL，视后端具体选型而定)
* 向量数据库 (如 ChromaDB/Milvus/Qdrant，用于 RAG 支持)

### 2. 配置环境变量
复制 `.env.example` 为 `.env` 文件，并填入必要的 LLM 密钥与基础配置


### 3. 运行项目
- 启动后端：在仓库根目录执行 `uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000`
- 启动前端：进入 frontend 目录执行 `npm install` 后运行 `npm run dev`
