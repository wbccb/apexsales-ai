# 销售 AI 闭环系统 MVP 概要技术设计方案

## 1. 需求目标与边界

### 1.1 目标
- 在浏览器侧完成语音分段采集与上传
- 后端完成语音转写与销售/客户二分声纹判别
- 前端以聊天流形式展示逐条对话
- 会后生成结构化 PRD 与报价摘要
- 生成可交互 POC Demo
- 一键导出合同 PDF

### 1.2 MVP 边界
- 不做纯流式逐字转写，采用 0.5 秒停顿切片
- 不做复杂多人聚类，采用销售声纹 1v1 对比
- 以可用性与速度优先，先跑通全链路

## 2. 技术选型

### 2.1 前端
- 框架：Next.js (App Router)
- UI：React + Tailwind CSS
- 语音端点检测：@ricky0123/vad-web
- POC 渲染：@codesandbox/sandpack-react
- 富文本编辑：TipTap 或 React-Quill

### 2.2 后端
- 框架：Python FastAPI
- ASR：FunASR + SenseVoice
- 声纹：Resemblyzer 或 FunASR CAM++
- 向量检索：PostgreSQL + pgvector
- 异步任务：FastAPI BackgroundTasks（MVP 优先）

### 2.3 部署
- 前端：Vercel
- 后端：阿里云 / 自建 GPU 服务器
- 数据库：Supabase（含 pgvector）

## 3. 关键链路设计

### 3.1 语音采集与切片
1. 前端使用 VAD 判断说话起止
2. 说话停顿 0.5 秒后打包音频为 audio/wav
3. POST 到后端 /asr/transcribe 接口

### 3.2 ASR 与声纹判别
1. ASR 将音频片段转写为文本
2. 声纹向量与销售基准向量进行余弦相似度对比
3. 相似度 > 0.8 标记为销售，否则为客户

### 3.3 会话记录与展示
1. 逐条保存 utterance
2. 前端以聊天流方式实时渲染

### 3.4 会后总结与 PRD 生成
1. 拼接会话全量文本
2. 调用 LLM 生成结构化 PRD 与报价摘要
3. 前端支持富文本编辑修订

### 3.5 POC 生成与渲染
1. PRD -> Prompt -> 生成单页 React 代码
2. Sandpack 即时渲染 POC
3. 生成可分享的短链

### 3.6 合同导出
1. PRD + 报价渲染合同模板
2. 导出 PDF

## 5. API 草案

### 5.1 声纹注册
- POST /voice/register
- 入参：音频文件
- 出参：声纹向量存储结果

### 5.2 语音转写 + 声纹判别
- POST /asr/transcribe
- 入参：音频文件、session_id
- 出参：{ speaker, text, ts }

### 5.3 会话记录
- GET /session/:id/utterances
- 出参：utterance 列表

### 5.4 会后总结
- POST /session/:id/summary
- 出参：PRD markdown

### 5.5 POC 生成
- POST /prd/:id/poc
- 出参：POC code

### 5.6 POC 分享
- GET /poc/:uuid
- 出参：POC code

### 5.7 合同导出
- POST /contract/:prd_id
- 出参：PDF 链接

## 6. 风险与权衡

- 声纹二分类仅适合销售/客户场景，不适合多人会议
- ASR 速度依赖 GPU 与音频切片长度
- LLM 输出不稳定，需要 Prompt 约束与人工校对

## 7. 里程碑建议

- 阶段一：跑通语音切片 + ASR + 对话流展示
- 阶段二：PRD 总结与可编辑
- 阶段三：POC 生成与 Sandpack 渲染
- 阶段四：合同导出与分享
