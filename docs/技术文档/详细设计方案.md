# 销售 AI 闭环系统 MVP 详细设计方案

## 1. 模块设计图（分层与调用链路）

### 1.1 前端分层

```
UI Layer
  - 会话聊天流
  - PRD 编辑器
  - POC 预览
  - 合同导出入口
Application Layer
  - 会话状态管理
  - 语音采集控制
  - 调用后端 API
Infrastructure Layer
  - VAD 语音切片
  - Sandpack 渲染
  - 文件上传与下载
```

### 1.2 后端分层

```
API Layer (FastAPI)
  - /voice/register
  - /asr/transcribe
  - /session/:id/summary
  - /prd/:id/poc
  - /contract/:prd_id
Service Layer
  - ASR 服务
  - 声纹比对服务
  - PRD 生成服务
  - POC 生成服务
  - 合同生成服务
Data Layer
  - PostgreSQL + pgvector
  - 对象存储（音频/合同）
```

### 1.3 语音链路调用流程

```
用户说话
  -> 前端 VAD 采集切片
  -> POST /asr/transcribe
      -> ASR 识别文本
      -> 声纹比对得到 speaker
      -> 保存 utterance
  -> 前端展示聊天流
```

### 1.4 会后总结链路

```
点击生成总结
  -> POST /session/:id/summary
      -> 获取全量对话
      -> 调用 LLM 生成 PRD
      -> 保存 PRD
  -> 前端展示可编辑 PRD
```

### 1.5 POC 生成链路

```
点击生成 POC
  -> POST /prd/:id/poc
      -> PRD -> Prompt -> LLM 生成前端代码
      -> 保存 POC 代码
  -> Sandpack 渲染 POC
```

### 1.6 合同导出链路

```
点击导出合同
  -> POST /contract/:prd_id
      -> PRD + 报价模板合成
      -> 生成 PDF
  -> 返回 PDF 下载链接
```

## 2. 数据库表结构 DDL

### 2.1 users

```sql
create table users (
  id uuid primary key,
  name text not null,
  role text not null,
  voice_embedding vector(256),
  created_at timestamptz default now()
);
```

### 2.2 sessions

```sql
create table sessions (
  id uuid primary key,
  user_id uuid references users(id),
  title text,
  created_at timestamptz default now()
);
```

### 2.3 utterances

```sql
create table utterances (
  id uuid primary key,
  session_id uuid references sessions(id),
  speaker text not null,
  text text not null,
  audio_url text,
  ts timestamptz default now()
);
```

### 2.4 documents

```sql
create table documents (
  id uuid primary key,
  name text not null,
  content text not null,
  embedding vector(1536),
  created_at timestamptz default now()
);
```

### 2.5 prds

```sql
create table prds (
  id uuid primary key,
  session_id uuid references sessions(id),
  markdown text not null,
  edited_markdown text,
  created_at timestamptz default now()
);
```

### 2.6 pocs

```sql
create table pocs (
  id uuid primary key,
  prd_id uuid references prds(id),
  code text not null,
  share_uuid text unique not null,
  created_at timestamptz default now()
);
```

### 2.7 contracts

```sql
create table contracts (
  id uuid primary key,
  prd_id uuid references prds(id),
  pdf_url text not null,
  created_at timestamptz default now()
);
```

## 4. Prompt 模板与评估策略

### 4.1 PRD 生成 Prompt

```
你是资深售前产品经理。
请基于以下会议逐字稿生成结构化 PRD，输出 Markdown：
1. 需求背景
2. 核心痛点
3. 业务流程
4. 功能清单
5. 交互草图描述
6. 报价建议
要求：条目化、可执行、避免编造。
会议逐字稿：
{{transcript}}
```

### 4.2 POC 生成 Prompt

```
你是资深前端工程师。
请将以下 PRD 转换为单页面 React 应用代码：
1. 使用 Tailwind CSS
2. 必须包含假数据
3. 不要写任何后端请求
4. 只输出代码
PRD：
{{prd}}
```

### 4.3 评估策略

#### 4.3.1 PRD 质量
- 完整性：是否覆盖需求背景、流程、功能、报价
- 可执行性：是否具备可落地的功能描述
- 一致性：是否与逐字稿一致，无明显臆造

#### 4.3.2 POC 质量
- 可运行：代码是否可在 Sandpack 直接渲染
- 对齐度：与 PRD 关键模块是否一致
- 交互性：是否具备最小可交互能力

#### 4.3.3 ASR 与声纹
- ASR 准确率：抽样评估逐字稿与音频一致性
- 声纹准确率：销售/客户二分正确率
