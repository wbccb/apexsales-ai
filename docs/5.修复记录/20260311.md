# 2026-03-11 修复记录

## 1. 问题现象

- `/asr/transcribe` 在部分环境返回 503
- 报错核心信息：`iic/SenseVoiceSmall is not registered`
- 前端无法直观看到当前 ASR 实际使用的引擎

## 2. 根因

- 当前 FunASR 默认模型在运行环境不可加载，导致转写链路抛异常
- 前端仅展示转写文本，没有引擎元信息，排障成本较高

## 3. 修复内容

### 3.1 后端

- 新增双引擎转写策略：`funasr` + `faster-whisper`
- 支持 `ASR_PROVIDER` 三种模式：
  - `funasr`
  - `faster_whisper`
  - `auto`（先 funasr，失败后自动切换 whisper）
- `/asr/transcribe` 返回新增字段：
  - `asr_engine`：`funasr` / `whisper` / `fallback`
  - `asr_fallback`：`true` / `false`
- 会话逐字稿内存项同步保存上述元信息

### 3.2 前端

- 主页面新增“ASR 当前引擎”实时显示
- 每条转写片段新增“引擎：xxx”标记
- 阶段状态中的 ASR 项改为动态显示当前引擎状态

## 4. 关键配置项

- `ASR_PROVIDER=auto`
- `ASR_FASTER_WHISPER_MODEL=tiny|small|medium|large-v3`
- `ASR_FASTER_WHISPER_DEVICE=cpu|cuda`
- `ASR_FASTER_WHISPER_COMPUTE_TYPE=int8|float16`
- `ASR_LANGUAGE=zh`
- `ASR_FALLBACK_TEXT=...`

## 5. 验证结果

- `python -m py_compile backend/app/main.py backend/app/config.py` 通过
- 前端 `npm run build` 通过
- 接口实测：
  - `GET /health` 返回 200
  - `POST /asr/transcribe` 返回 200，且包含 `asr_engine` 与 `asr_fallback`
