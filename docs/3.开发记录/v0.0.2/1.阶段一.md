# 阶段一开发
## 任务拆解
- 任务 1：新增知识库文档上传接口 `POST /knowledge/upload`
  - 输入 PDF 文件，保存文档元信息，初始状态置为 `pending`
  - 上传成功后触发后台索引任务
- 任务 2：新增知识库文档列表接口 `GET /knowledge/documents`
  - 支持按 `status`、`owner_user_id` 过滤
  - 支持分页参数 `page/page_size`
- 任务 3：新增文档重建索引接口 `POST /knowledge/reindex/{document_id}`
  - 支持失败文档重试与已完成文档手动重建
  - 状态重置为 `pending` 并重新触发索引
- 任务 4：新增检索接口 `POST /knowledge/retrieve`
  - 输入 `query/top_k/business_tag`
  - 输出 TopK 片段与来源字段（document_id/chunk_id/page_no）
- 任务 5：实现阶段一状态机与内存数据结构
  - 文档状态：`pending -> processing -> ready | failed`
  - 维护文档缓存与切片缓存，支持后续阶段二 summary 接入
- 任务 6：补齐依赖与基础校验
  - 增加 PDF 读取依赖
  - 执行 Python 语法编译校验确保改动可运行

## 实现功能与代码逻辑

- 已实现阶段一全部任务（P0-RAG 基础链路）
- 后端新增知识库数据模型与响应模型
  - 新增 `KnowledgeUploadResponse`、`KnowledgeDocumentItem`、`KnowledgeDocumentsResponse`、`KnowledgeReindexResponse`、`KnowledgeRetrieveRequest`、`KnowledgeMatch`、`KnowledgeRetrieveResponse`

- 后端新增四个阶段一接口
  - `POST /knowledge/upload`
  - `GET /knowledge/documents`
  - `POST /knowledge/reindex/{document_id}`

  - 代码位置：`backend/app/main.py`
- 阶段一状态机已落地
  - 上传后 `pending`
  - 后台任务执行 `processing`
  - 成功进入 `ready`
  - 失败进入 `failed` 并记录 `error_message`

  - 结果：通过
- 阶段一联调验证结果（真实 PDF 全流程）
  - 验证方式：使用 `reportlab` 生成真实 PDF（含报价文本），通过 `TestClient` 依次调用 upload/list/retrieve/reindex
  - 执行命令：`. .venv-validation/bin/activate && python .tmp_stage1_validate.py`（已执行并清理临时脚本）
  - upload：`POST /knowledge/upload` 返回 `200`，文档状态初始为 `pending`
  - list：`GET /knowledge/documents` 查询到同一文档状态变为 `ready`，`chunk_count=1`
  - retrieve：`POST /knowledge/retrieve` 返回 `200`，命中 `1` 条片段，返回内容包含 `Pro plan price 59999`
  - reindex：`POST /knowledge/reindex/{document_id}` 返回 `200` 且状态回执为 `pending`，随后 list 再次查询状态为 `ready`
  - 关键返回（节选）：
    - `upload ... {"status_code": 200, "body": {"status": "pending", ...}}`
    - `list_after_upload ... {"status": "ready", "chunk_count": 1}`
    - `retrieve ... {"status_code": 200, "match_count": 1, "top_match_preview": "ApexSales Product and Quote ... Pro plan price 59999 ..."}`
    - `reindex ... {"status_code": 200, "body": {"status": "pending"}}`
