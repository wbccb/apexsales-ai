# 阶段一开发
## 任务拆解

### 阶段一拆解（具体任务清单）
- 前端语音切片采集
- 前端上传到后端并处理响应
- 前端聊天流展示
- 后端 /asr/transcribe 接口
- 后端声纹 1v1 判别
- 后端 utterance 保存（阶段一仅需内存或轻量存储即可）
  
### 接口清单（阶段一最小可用）
- POST /asr/transcribe
- 入参：audio 文件（wav）、session_id（先可用固定值）
- 出参：{ speaker: "销售" | "客户", text: string, ts: ISO }
- 后端内部流程：ASR → 声纹比对 → 生成 utterance

### 前端模块清单（基于当前结构）
- 页面入口： page.tsx
- 语音采集模块（VAD 控制器）
- 上传模块（audio → API）
- 聊天流渲染组件（列表 + 气泡）
- 会话状态管理（session_id、utterances）

### 按当前仓库结构的执行计划
- backend/app/main.py：新增 /asr/transcribe 路由与处理流程
- backend/requirements.txt：补齐 ASR/声纹依赖（先确认可用库再加）
- frontend/src/app/page.tsx：添加录音按钮、VAD 触发、上传逻辑、聊天渲染
- frontend/src/app/globals.css：补充聊天 UI 基础样式（如气泡/布局）

### 阶段一分步落地顺序
- Step 1：后端 /asr/transcribe 返回假数据（保证链路通）
- Step 2：前端 VAD 采集与上传，聊天流接入假数据
- Step 3：后端接入真实 ASR（SenseVoice 或替代）
- Step 4：后端接入声纹 1v1 判别
- Step 5：前端展示真实识别结果

## 阶段一实现情况
- 已全部实现（前端 VAD/上传/聊天流 + 后端 ASR/声纹/utterance 记录）

## 实现功能与代码逻辑
- 前端语音切片采集
  - 使用 @ricky0123/vad-web 的 MicVAD 实例，onSpeechStart/onSpeechEnd 监听说话开始与结束
  - onSpeechEnd 收到 Float32Array 音频后进行 WAV 编码并触发上传
  - 代码位置：frontend/src/app/page.tsx
- 前端上传到后端并处理响应
  - encodeWav 将 Float32Array 转为 16kHz 单声道 wav
  - FormData 组装 audio、session_id、sales_id 后调用 /asr/transcribe
  - 返回 utterance_id/speaker/text/ts 追加到页面对话列表
  - 代码位置：frontend/src/app/page.tsx
- 前端聊天流展示
  - utterances 状态存储聊天列表并按时间渲染
  - UI 直接展示 speaker、时间、文本内容
  - 代码位置：frontend/src/app/page.tsx
- 后端 /asr/transcribe 接口
  - 接收 audio、session_id、sales_id
  - 音频保存到临时文件后执行 ASR 转写
  - 输出 utterance_id、speaker、text、ts
  - 代码位置：backend/app/main.py
- 后端真实 ASR
  - 使用 FunASR AutoModel 加载 iic/SenseVoiceSmall
  - generate 返回文本后执行 rich_transcription_postprocess
  - 代码位置：backend/app/main.py
- 后端声纹 1v1 判别
  - /voice/register 保存 sales 声纹向量
  - /asr/transcribe 中取 sales embedding 计算余弦相似度
  - similarity >= 配置阈值 判定为销售，否则客户
  - 代码位置：backend/app/main.py
- 后端 utterance 保存
  - 以 session_id 为键保存 utterance 列表（内存态）
  - 代码位置：backend/app/main.py
- 配置下沉
  - ASR 模型、设备、声纹阈值统一从 config 读取
  - 代码位置：backend/app/config.py
