# 阶段二开发
阶段二目标
- 会后总结 /session/:id/summary
- PRD 生成与可编辑
- 基础数据落库（sessions/utterances/prds）
- UI 展示 PRD 与编辑结果

阶段二任务分解
- 后端：会话与逐字稿聚合
  - 提供 session 维度的 utterances 查询与合并逻辑
  - 输出完整 transcript 给 LLM
- 后端：PRD 生成服务
  - 实现 /session/:id/summary 接口
  - Prompt 结构化输出 Markdown
  - 保存 prd 记录（markdown + edited_markdown）
- 后端：数据存储
  - 落库 sessions/utterances/prds
  - 为 summary 生成与编辑保存添加 DAO 或简单存储层
- 前端：PRD 展示与编辑
  - 新增 PRD 预览区域
  - 支持编辑与保存 edited_markdown
- 前后端联调与验收
  - 点击“生成总结”触发 summary 接口
  - 渲染 PRD，手动修改后保存

阶段二接口草案
- GET /session/:id/utterances
  - 返回 utterance 列表
- POST /session/:id/summary
  - 入参：session_id
  - 出参：{ prd_id, markdown }
- POST /prd/:id/save
  - 入参：edited_markdown
  - 出参：保存结果

阶段二验收标准
- summary 接口可生成结构化 PRD（Markdown）
- 前端可编辑并保存 PRD
- 会话逐字稿可从后端完整拉取并展示

## 阶段二实现情况
- 已全部实现（规则版 PRD + 可切换 LLM 版 + 前端编辑保存）

## 实现功能与代码逻辑
- 会话逐字稿聚合与查询
  - /session/{session_id}/utterances 直接读取内存态 utterances_by_session 并返回列表
  - 逐字稿拼接通过 build_transcript 生成 [speaker] 文本序列
  - 代码位置：backend/app/main.py
- PRD 生成服务
  - /session/{session_id}/summary 拉取会话 utterances 并生成 transcript
  - generate_prd_markdown 根据 PRD_MODE 在规则版与 LLM 版之间切换
  - 代码位置：backend/app/main.py
- 规则版 PRD
  - generate_prd_markdown_rule 按最小结构模板输出 Markdown
  - 代码位置：backend/app/main.py
- LLM 版 PRD
  - generate_prd_markdown_llm 读取 LLM_API_URL/LLM_MODEL/LLM_API_KEY 等配置
  - Prompt 从 config.get_prd_prompt_template 读取
  - 代码位置：backend/app/main.py + backend/app/config.py
- PRD 保存
  - /prd/{prd_id}/save 将 edited_markdown 保存到内存态 prds_by_id
  - 代码位置：backend/app/main.py
- 前端 PRD 展示与编辑
  - 点击“生成总结”调用 /session/{session_id}/summary 返回 prd_id + markdown
  - textarea 支持编辑 edited_markdown 并提交保存
  - 代码位置：frontend/src/app/page.tsx
