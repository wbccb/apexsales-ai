# 自测清单

## 0. 环境准备
- 启动后端：在仓库根目录执行 `uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000`
- 启动前端：进入 frontend 目录执行 `npm install` 后运行 `npm run dev`
- 默认前端调用后端地址为 `http://localhost:8000`，如需修改请设置 `NEXT_PUBLIC_API_BASE`

## 1. 健康检查 /health
- 操作：`curl http://localhost:8000/health`
- 验证：返回 `{"status":"ok"}`

## 2. 声纹注册 /voice/register
- 操作：
  - `curl -F user_id=sales_001 -F audio=@/path/to/sales.wav http://localhost:8000/voice/register`
- 验证：
  - 返回 `voice_embedding_saved: true`
  - 后续声纹验证可通过同一 user_id 校验

## 3. 声纹验证 /voice/verify
- 操作：
  - `curl -F user_id=sales_001 -F audio=@/path/to/sales.wav http://localhost:8000/voice/verify`
  - 使用非销售音频再测一次
- 验证：
  - 相同音频时 `is_sales` 为 `true`
  - 不同音频时 `is_sales` 为 `false`
  - `similarity` 数值为 0~1

## 4. ASR 转写 + 声纹二分 /asr/transcribe
- 操作：
  - `curl -F session_id=session_001 -F sales_id=sales_001 -F audio=@/path/to/sales.wav http://localhost:8000/asr/transcribe`
  - 使用客户音频再测一次
- 验证：
  - 返回 `text` 为非空转写文本
  - `speaker` 为“销售/客户”二选一
  - 同一会话多次调用可持续追加 utterance

## 5. 会话记录 /session/{id}/utterances
- 操作：`curl http://localhost:8000/session/session_001/utterances`
- 验证：
  - utterances 列表数量随转写增加
  - 每条包含 `id/speaker/text/ts`

## 6. 会后总结 /session/{id}/summary
- 操作：
  - `curl -X POST http://localhost:8000/session/session_001/summary`
  - 或在前端点击“生成总结”
- 验证：
  - 返回 `prd_id` 和 `markdown`
  - markdown 至少包含“需求背景/核心痛点/功能清单”等段落

## 7. PRD 保存 /prd/{prd_id}/save
- 操作：
  - `curl -X POST http://localhost:8000/prd/{prd_id}/save -H "Content-Type: application/json" -d '{"edited_markdown":"# 自测 PRD\\n- 修改内容"}'`
  - 或在前端编辑后点击保存
- 验证：
  - 返回 `saved: true`
  - 之后生成 POC/合同时使用编辑后的内容

## 8. POC 生成 /prd/{prd_id}/poc
- 操作：
  - `curl -X POST http://localhost:8000/prd/{prd_id}/poc`
  - 或在前端点击“生成 POC”
- 验证：
  - 返回 `poc_id/code/share_uuid`
  - 前端 Sandpack 可渲染页面

## 9. POC 分享 /poc/{share_uuid}
- 操作：
  - `curl http://localhost:8000/poc/{share_uuid}`
  - 或在前端输入 share_uuid 点击“打开分享”
- 验证：
  - 返回 code 与生成时一致
  - Sandpack 能复现同一 POC

## 10. 合同生成 /contract/{prd_id}
- 操作：
  - `curl -X POST http://localhost:8000/contract/{prd_id}`
  - 或在前端点击“导出合同”
- 验证：
  - 返回 `contract_id` 与 `pdf_url`
  - 浏览器可打开 PDF

## 11. 合同下载 /contract/{contract_id}/download
- 操作：
  - `curl -O http://localhost:8000/contract/{contract_id}/download`
- 验证：
  - 成功下载 PDF 文件
  - 文件内容与 PRD 文本一致

## 12. 模板与样式配置
- 操作：
  - 修改 `backend/app/templates/poc_rule_template.txt` 后重新生成 POC
  - 修改 `backend/app/templates/contract_template.txt` 或 `backend/app/templates/contract_style.json` 后重新生成合同
  - 需要自定义路径时设置 `POC_RULE_TEMPLATE_PATH` / `CONTRACT_TEMPLATE_PATH` / `CONTRACT_STYLE_PATH`
- 验证：
  - POC code 中出现新的标题或布局
  - 合同 PDF 中标题字体/边距/行距按样式生效
