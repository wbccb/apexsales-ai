# 🚀 销售 AI 闭环系统 —— v0.0.1 需求文档（MVP）

## 一、文档目标
- 定义 v0.0.1 的最小可用产品需求，用于指导开发、联调与验收。
- v0.0.1 只覆盖“语音采集 -> 转写与说话人识别 -> 会后总结 -> POC 预览 -> 合同导出”的主闭环。
- v0.0.1 以快速可演示为目标，不追求大规模并发与复杂企业级能力。

## 二、MVP 原则
- 不做逐字级纯流式识别，采用“回合制伪实时”。
- 以前端语音端点检测触发切片上传，降低后端流式复杂度。
- 声纹阶段采用“销售 vs 客户”的 1v1 判别方案，优先可用性。
- LLM 输出允许人工编辑校正，确保业务可控。

## 三、范围定义

### In Scope（v0.0.1 必须交付）
- 前端语音采集与切片上传。
- 后端 ASR 中文转写与销售/客户二分。
- 会话逐字稿查询与会后 PRD 总结生成。
- PRD 在线编辑与保存。
- PRD 转前端 POC 代码并在 Sandpack 渲染。
- POC 分享码读取能力。
- 合同 PDF 生成与下载。

### Out of Scope（v0.0.1 不包含）
- RAG 知识库（PDF 向量化、报价检索注入）。
- 真实数据库持久化（允许内存态先跑通）。
- 客户“打开短链即自动展示”的独立分享页体验。
- 完整部署方案与生产级运维能力。

## 四、功能需求

### FR-01 语音采集与上传
- 用户可开启/暂停麦克风监听。
- 说话片段结束后，前端将音频编码为 `audio/wav` 上传后端。
- 上传时需携带会话标识 `session_id`。

### FR-02 ASR 转写与说话人识别
- 系统提供音频转写接口，返回文本与说话人标签。
- 系统支持销售声纹注册与验证。
- 当提供销售身份时，系统按相似度阈值输出“销售/客户”标签。

### FR-03 会话逐字稿与总结
- 系统可按 `session_id` 获取会话逐字稿。
- 系统可根据逐字稿生成结构化 Markdown 总结（PRD 草稿）。
- 总结结果应可被后续编辑与保存。

### FR-04 PRD 编辑保存
- 前端需提供 PRD 可编辑区域。
- 用户可保存编辑后的 PRD 内容。

### FR-05 POC 生成与预览
- 系统可根据 PRD 生成可运行的前端代码字符串。
- 前端需在 Sandpack 中即时渲染该代码。
- 系统支持通过分享 UUID 拉取同一份 POC 代码。

### FR-06 合同导出
- 系统可基于 PRD 内容生成合同 PDF。
- 系统返回可下载地址，前端支持直接打开或下载。

## 五、非功能需求
- 响应速度：MVP 场景下，单段语音转写应具备可交互体验。
- 稳定性：主要接口具备基础错误返回与失败提示。
- 可配置性：ASR 模型、声纹阈值、LLM 参数支持环境配置。
- 安全性：不在日志中输出密钥；下载接口需做资源存在性校验。

## 六、接口需求（v0.0.1）
- `GET /health`
- `POST /voice/register`
- `POST /voice/verify`
- `POST /asr/transcribe`
- `GET /session/{session_id}/utterances`
- `POST /session/{session_id}/summary`
- `POST /prd/{prd_id}/save`
- `POST /prd/{prd_id}/poc`
- `GET /poc/{share_uuid}`
- `POST /contract/{prd_id}`
- `GET /contract/{contract_id}/download`

## 七、验收标准
- 可完成“采集语音 -> 生成聊天记录 -> 生成 PRD -> 生成 POC -> 导出合同”的全链路演示。
- 关键接口按约定返回结构化结果，前端能正确展示。
- 失败场景有明确提示，不出现静默失败。
- 产出物可复用：PRD 可保存，POC 可通过 UUID 读取，合同可下载。
