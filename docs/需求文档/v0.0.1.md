# 🚀 销售 AI 闭环系统 —— MVP 极速版需求与技术文档

## 一、 核心痛点与妥协原则 (MVP 铁律)
*   **妥协点：** 不做逐字蹦出的纯流式声纹识别。
*   **实现方式：** 采用“回合制”伪实时。当某人说完一句话停顿（0.5秒）时，系统立刻将这句话转成文字并打上说话人标签。这对销售复盘和提取需求来说**完全够用**。

## 二、 GitHub 开源库选型（MVP 技术栈）

### 1. 前端语音端点检测 (VAD) —— 解决“什么时候该切断录音”
*   **推荐库：`ricky0123/vad`** (Browser-based Silero VAD)
*   **GitHub 地址：**[https://github.com/ricky0123/vad](https://github.com/ricky0123/vad)
*   **为什么用它：** 这是一个可以在**纯浏览器前端**运行的 VAD（语音活动检测）库。它能极其精准地判断用户“什么时候开始说话，什么时候闭嘴”。
*   **MVP 玩法：** 销售和客户说话时，前端自动监听。只要有人闭嘴超过 0.5 秒，前端立刻把刚才那段声音打包成 `audio/wav` 扔给后端 API。**这直接省去了后端处理复杂音频流的恶梦。**

### 2. 后端 ASR (语音转文字) —— 解决“极速且免费的中文转录”
*   **推荐库：`FunASR`** (阿里达摩院开源) / 具体使用 **`SenseVoice`** 模型
*   **GitHub 地址：**[https://github.com/alibaba-damo-academy/FunASR](https://github.com/alibaba-damo-academy/FunASR) 或[https://github.com/FunAudioLLM/SenseVoice](https://github.com/FunAudioLLM/SenseVoice)
*   **为什么用它：** `SenseVoice` 是阿里最新开源的基础语音模型，**速度比原版 Whisper 快 50 倍**，而且对中文、粤语、甚至中英夹杂的识别极好。
*   **MVP 玩法：** 用 Python (FastAPI) 写一个极简接口，接收前端传来的音频切片，调用 `SenseVoice`，几十毫秒就能返回这段话的文字。

### 3. 后端声纹区分 (Diarization) —— 解决“谁在说话”的极简 Hack 方案
*   **推荐库：`Resemblyzer`** 或 FunASR 内置的 **`CAM++`**
*   **GitHub 地址：**[https://github.com/resemble-ai/Resemblyzer](https://github.com/resemble-ai/Resemblyzer) (极其简单易用的 Python 声纹库)
*   **为什么用它：** 传统聚类算法太慢太复杂。我们在 MVP 阶段使用**“声纹 1v1 比对法”**。
*   **MVP 玩法（声纹魔法）：**
    1. 销售在注册账号时，对着系统念一段话（提取并保存为“销售特征向量 A”）。
    2. 开会时，前端传来的每一段音频，后端都用 `Resemblyzer` 提取一次特征，并与“销售特征 A”计算相似度（就一行代码：计算余弦距离）。
    3. **相似度 > 80%，标记为 [销售]；相似度 < 80%，标记为 [客户]。** 完美绕开复杂的多人声纹聚类！

### 4. 前端 UI POC 生成 —— 解决“下午看 Demo”
*   **推荐库：`sandpack-react`**
*   **GitHub 地址：** [https://github.com/codesandbox/sandpack](https://github.com/codesandbox/sandpack)
*   **为什么用它：** 直接在浏览器里运行 LLM 生成的 React 代码，不需要你起后台 Node.js 服务器，前端即插即用。

## 三、 14天开发计划

### 🟢 第 1-4 天：跑通“伪实时”语音识别链路 (最硬核的部分)
*   **Day 1 (前端 VAD)：**
    *   使用 Next.js 搭建前端。
    *   引入 `@ricky0123/vad-web`。实现页面上有一个麦克风按钮，点击后开始监听，一旦说话人停顿，就在 Console 里打印出一个 Audio Blob。
*   **Day 2 (后端 ASR 接口)：**
    *   起一个 Python FastAPI 项目。
    *   拉取 `SenseVoice` 或使用极简版的 `Faster-Whisper`，写一个接收音频文件并返回文字的 POST 接口。
*   **Day 3 (声纹 Hack)：**
    *   在 FastAPI 中集成 `Resemblyzer`。
    *   写一个“预注册”接口保存销售的基准声音。
    *   修改 Day 2 的接口：收到音频 -> 转成文字 + 计算声纹相似度 -> 返回 `{"speaker": "销售/客户", "text": "刚刚说的话"}`。
*   **Day 4 (全链路联调)：**
    *   前后端打通。实现效果：销售把手机放在桌子上，两人对话，手机屏幕上像微信聊天一样，一段一段地弹出“[销售]：你好” 和 “[客户]：你们这怎么卖”。

### 🟡 第 5-8 天：RAG 知识库与大模型“大脑”注入
*   **Day 5 (文档切片)：** 使用 `LangChain` 和 Supabase 的 `pgvector`，实现上传 PDF 文档（公司产品介绍、报价单）并存为向量。
*   **Day 6-7 (Prompt 链组装)：**
    *   会议结束，点击“生成总结”。
    *   将前端收集到的完整聊天记录（逐字稿）拼起来。
    *   写 Prompt 让大模型（如 DeepSeek-V3 或 Claude 3.5）提取需求点、匹配 RAG 库里的报价，生成一份结构化的 Markdown PRD 文档。
*   **Day 8 (前端展示与编辑)：** 在 UI 上展示生成的 PRD 和报价，允许销售像编辑富文本一样手动修改大模型瞎编的地方（Human-in-the-loop，防锅机制）。

### 🔴 第 9-12 天：AI 生成 POC（魔法时刻）
*   **Day 9 (Sandpack 集成)：** 在前端项目里引入 `<Sandpack />` 组件，跑通最基本的 React 渲染。
*   **Day 10 (代码生成 Prompt)：**
    *   写一个极其严苛的 Prompt 给大模型：“你现在要把上面的 PRD 转换为一个纯前端的单页面 React 应用。使用 Tailwind 样式，必须包含假数据，不要写任何后端请求。只输出代码。”
*   **Day 11 (联调渲染)：** 将大模型返回的代码字符串，自动填入 `Sandpack` 中，实现页面左侧是需求，右侧直接渲染出可点击的系统界面 Demo。
*   **Day 12 (分享链接)：** 将这段代码存入数据库，生成一个 UUID 短链。销售下午可以直接把短链发给客户，客户手机打开直接看到高保真 UI。

### 🟣 第 13-14 天：合同生成与收尾
*   **Day 13 (一键合同)：**
    *   引入 GitHub 库 `react-pdf/renderer` 或直接用 HTML 转 PDF 工具。
    *   把 PRD 和报价单嵌进公司的标准合同模板里，一键导出 PDF。
*   **Day 14 (部署上线)：**
    *   前端部署到 Vercel (国际) 或 Zeabur (国内)。
    *   后端 Python API 部署到阿里云 Serverless 或自己买个带入门级显卡（如 RTX 3060 / 4060）的便宜云服务器。
