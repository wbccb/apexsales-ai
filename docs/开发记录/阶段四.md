# 阶段四开发

阶段四目标
- 合同生成与导出
- 合同模板管理
- 分享与下载链接

阶段四任务分解
- 后端：合同生成服务
  - 实现 /contract/:prd_id 接口
  - 合并 PRD/报价与合同模板生成 PDF
  - 保存合同记录与 pdf_url
- 后端：模板与报价数据
  - 维护标准合同模板
  - 结构化报价字段映射到模板
- 前端：合同导出入口
  - 在 PRD 页面提供“导出合同”按钮
  - 下载或打开 pdf_url
- 前后端联调与验收
  - 生成合同 PDF 并可下载
  - 分享链接可访问

阶段四接口草案
- POST /contract/:prd_id
  - 出参：{ contract_id, pdf_url }

阶段四验收标准
- PDF 合同可生成并下载
- 合同内容正确映射 PRD 与报价

阶段四实现情况
- 已全部实现（后端合同生成 + 前端导出入口）

实现功能与代码逻辑
- 后端：合同生成服务
  - /contract/{prd_id} 读取 PRD 生成 PDF，并返回下载链接
  - render_contract_pdf 读取合同模板与样式配置后输出 PDF
  - 默认模板文件：backend/app/templates/contract_template.txt
  - 默认样式文件：backend/app/templates/contract_style.json
  - 模板路径可通过 CONTRACT_TEMPLATE_PATH 覆盖
  - 样式路径可通过 CONTRACT_STYLE_PATH 覆盖
  - 代码位置：backend/app/main.py + backend/app/config.py
- 后端：合同下载
  - /contract/{contract_id}/download 返回 PDF 文件
  - 代码位置：backend/app/main.py
- 前端：合同导出入口
  - “导出合同”调用 /contract/{prd_id} 并打开 pdf_url
  - 代码位置：frontend/src/app/page.tsx
