# 阶段二开发
## 任务拆解
- 任务 1：改造 `POST /session/{session_id}/summary` 入参与返回结构
  - 新增可选入参：`rag_enabled`、`top_k`、`business_tag`
  - 保留兼容字段：`prd_id`、`markdown`
  - 新增返回字段：`citations`、`rag_used`
- 任务 2：实现 summary 的 RAG 注入链路
  - 用 transcript 作为 query 检索知识库 TopK
  - 构造 `retrieved_context` 注入 PRD 生成输入
  - 命中片段生成可追溯 citations
- 任务 3：落地 citations 持久化基础结构
  - PRD 记录补充 `rag_enabled` 元数据
  - 新增 `prd_citations` 数据存储与查询能力
- 任务 4：前端接入 summary 增强能力
  - 生成总结时传递 `rag_enabled/top_k`
  - 展示 `rag_used` 与 citations 列表
- 任务 5：联调与回归
  - 验证无 body 请求仍兼容
  - 验证有 body 请求返回 citations 且结构稳定
