# 阶段三开发
阶段三目标
- PRD -> POC 代码生成
- POC Sandpack 渲染
- 生成分享短链

阶段三任务分解
- 后端：POC 生成服务
  - 实现 /prd/:id/poc 接口
  - 将 PRD 转为前端代码字符串（LLM Prompt）
  - 保存 code 与 share_uuid
- 后端：POC 分享
  - 实现 /poc/:uuid 接口
  - 根据 share_uuid 返回 POC code
- 前端：POC 预览区
  - 接入 Sandpack 渲染
  - PRD -> 触发生成 POC
- 前后端联调与验收
  - 点击“生成 POC”返回可运行代码
  - POC 可在 Sandpack 内实时展示
  - 分享链接可通过 uuid 打开

阶段三接口草案
- POST /prd/:id/poc
  - 出参：{ poc_id, code, share_uuid }
- GET /poc/:uuid
  - 出参：{ poc_id, code }

阶段三验收标准
- POC 代码可在 Sandpack 中直接渲染
- 分享短链可复现同一 POC

阶段三实现情况
- 已全部实现（后端 POC 生成/分享 + 前端 Sandpack 渲染）
  
实现功能与代码逻辑
- 后端：POC 生成服务
  - /prd/{prd_id}/poc 读取 PRD 后生成前端代码并保存 share_uuid
  - 规则版通过 generate_poc_code_rule 读取可配置模板文件并替换 {{prd}}
  - 默认模板文件：backend/app/templates/poc_rule_template.txt
  - 模板路径可通过 POC_RULE_TEMPLATE_PATH 覆盖
  - LLM 版通过 generate_poc_code_llm 调用模型生成代码
  - 代码位置：backend/app/main.py + backend/app/config.py
- 后端：POC 分享
  - /poc/{share_uuid} 根据 share_uuid 返回 POC code
  - 代码位置：backend/app/main.py
- 前端：POC 预览区
  - “生成 POC”调用 /prd/{prd_id}/poc 并回填 Sandpack
  - “打开分享”通过 /poc/{share_uuid} 拉取并渲染
  - 代码位置：frontend/src/app/page.tsx
